Consejos Git
    - Siempre hay que pasar los archivos al area de staging antes de hacer commit
    - La clave SSH se guarda en usuarios, se puede acceder rapido abriendo el git bash y usando cd ".ssh"
        despues de usar "ls", la clave de la izquierda es la privada, la de la derecha la publicagit
    - Para recuperar los archivos desde github hay que darle al boton de code en la pagina de hithub y copiar la linea de SSH 
Comandos terminal
    Ctrl + L --> limpiar terminal
Comandos Linux
    pwd --> muestra la ubicacion desde donde se abrio la terminal
    ls --> muestra los archivos que se encuentran en la carpeta en la que estas
        -a --> muestra tambien archivos ocultos
    rm "nombre-archivo" --> borra el archivo
    cd "nombre-archivo" --> se mueve hacia ese archivo
        cd --> vuelve entre los archivos (va hacia el directorio padre)
    "." --> hace referencia al directorio actual    
    ".." --> hace referencia al directorio padre
    -v --> muestra informacion mas detallada
    cat --> mostrar el contenido de un archivo
Comandos Git
    git ..
        init --> iniciar repositorio de git (carpeta administrada por git)
        status --> muestra el estado del repositorio
            "Untracked files" --> archivos que no estan siendo manejados por git (sin staging)
        add "nombre-archivo" --> pasar el archivo al area de staging (archivo manejado por git)
            add . --> agrega todos los archivos del directorio
        commit -m "comentario" --> confirma la actualizacion de todos los archivos en el area de staging
        log --> ver informacion sobre los commits
        checkout "id-commit o nombre-rama" --> regresa o avanza entre versiones de los commit (osea, entre commit hechos antes o despues del actual)
            checkout master --> vuelve al ultimo commit
        checkout -b "nombre-rama" --> crea una rama y se mueve hacia ella
            -b -->  hace referencia a branch (rama)
        branch --> ver las ramas
        reset "id-commit" --> vuelve al commit especificado, borrando los posteriores (sin contar la carpeta en la que se trabaja)
            reset --hard "id-commit" --> vuelve al commit especificado, borrando los posteriores (incluyendo los cambios en la carpeta)
        --oneline --> muestra la informacion en una sola linea
        restore "nombre-archivo"--> recupera un archivo desde git hacia el directorio
        git remote add origin "url-repositorio" --> conecta el repositorio local con el de github
            remote --> hace referencia al repositorio remoto (repositorio de github)
            origin --> nombre del servidor, usado para resumir el "url-repositorio"
        push --> sube el repositorio a github
            -u origin master --> en caso de ser la primera vez, se indica la rama que se quiere crear
                -u = --set-upstream
                -u origin "nombre-rama" --> se crea una rama en el repositorio remoto y se mueven los archivos hacia ella
